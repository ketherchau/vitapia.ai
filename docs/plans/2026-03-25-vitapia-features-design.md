# Vitapia Core Features Implementation Plan

> **For Gemini:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement a global notification modal, websockets for real-time status updates, comprehensive simulation reports on the result page, and fix the PDF download button.

**Architecture:** 
- **Notifications:** A global React Context provider (`NotificationContext`) wrapping the application. It will render a beautiful TailwindCSS modal that replaces all `window.alert` calls.
- **Websockets:** The `vitapia_orchestrator` (Express) will be upgraded with `socket.io` to emit `simulation_ready` events when tasks finish. `vitapia_web` will use `socket.io-client` at the layout level to listen and trigger the notification modal.
- **Reports:** The `results/[id]/page.tsx` will be expanded to render a structured "Comprehensive Report" (Executive Summary, Methodology, etc.), leveraging the `final_report` generated by the orchestrator.
- **PDF Download:** The "Download PDF" button will be updated to point to the existing Next.js API route (`/api/simulations/[id]/pdf`) via an `<a>` tag instead of calling `window.print()`.

**Tech Stack:** Next.js App Router, React Context, TailwindCSS, Socket.io, PDFKit.
